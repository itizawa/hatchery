# 設計書: /design-review スキルにデザイン観点チェックリストを追加する (#794)

## 1. 目的 / 背景

現在の `/code-review` スキルはコードの正確性・バグ・効率性に特化しており、UI/デザイン品質の観点が含まれていない。
そのため PR レビューでデザインの退行（Generic スタイルの混入・スペーシング崩れ・インタラクション状態の欠落等）が見逃されやすい。

本 Issue では、client ワークスペースの UI 変更を含む PR のレビュー時にデザイン観点のチェックリストを自動的に適用するための
専用スラッシュコマンド `/design-review` を追加する。

## 2. スコープ（やること / やらないこと）

**やること**
- `.claude/commands/design-review.md` を新設し、デザイン観点チェックリストを定義する
- 既存の `/code-review` スキルは変更しない（追加・補完の位置付け）

**やらないこと**
- デザイン観点の自動修正（チェックのみ）
- Storybook へのデザインテスト追加
- `/code-review` の既存チェック項目の変更・削除

## 3. 受け入れ条件（テストに落とせる粒度で箇条書き）

1. `.claude/commands/design-review.md` が存在する
2. フロントマターに `description` フィールドがある
3. タイポグラフィ階層（サイズ・ウェイトのコントラスト）のチェック項目が含まれる
4. ホワイトスペースの一貫性（8px グリッド基準）のチェック項目が含まれる
5. カラー使用（アクセントの乱用・低コントラスト）のチェック項目が含まれる
6. インタラクション状態（hover / focus / active / disabled）のチェック項目が含まれる
7. 空状態・ローディング状態のチェック項目が含まれる
8. CLAUDE.md のデザインシステム方針との整合性チェックが含まれる
9. 既存の `/code-review` と併用できる旨が明記されている
10. `pnpm test:repo` が緑

## 4. 設計方針（アーキ・データ構造・主要モジュール）

### ファイル形式

既存コマンド（`df.md`, `goal.md` 等）と同じフォーマットで実装:

```
---
description: <1行説明>
argument-hint: <引数ヒント（省略可）>
allowed-tools: <許可ツール一覧>
---

# /design-review — コマンド本文（Markdown）
```

### チェックリスト構成

Patrick Neeman の UI レビュープロンプト知見と CLAUDE.md `## デザインシステム` 節を組み合わせ、
以下の6観点でチェックリストを構成する:

1. **タイポグラフィ階層** — サイズ・ウェイトのコントラスト、推奨フォント（Plus Jakarta Sans / DM Sans / Geist）の使用
2. **ホワイトスペース** — 8px グリッド基準の一貫したスペーシング
3. **カラー** — SLACK_COLORS の4色基盤を逸脱していないか、アクセントの乱用（1画面に3色以上）がないか
4. **インタラクション状態** — hover / focus / active / disabled の全状態が実装されているか
5. **空状態・ローディング状態** — 各コンテンツエリアに空状態・スケルトン等が存在するか
6. **CLAUDE.md デザインシステム整合性** — 禁止パターン（白紫グラデーション・汎用カードシャドウ等）に違反していないか
