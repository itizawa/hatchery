# 設計書: /df の自動選択を直近マイルストーンのみに限定し、対象が無ければスキップする (#754)

## 1. 目的 / 背景

引数なし `/df` の自動選択ロジックは、現状「open かつ `milestone/*` ラベル付きの Issue 全件」を候補とし、マイルストーン昇順でソートして 1 件選ぶ。このため直近マイルストーンが消化し切れると、自動的に次のマイルストーンへフォールバックしてしまう。

「いま締切が近い 1 マイルストーンだけを集中して回す」運用に合わせ、直近マイルストーン以外はフォールバック対象から除外し、対象 Issue が無ければスキップして終了するよう変更する。

## 2. スコープ（やること / やらないこと）

### やること
- `.claude/commands/df.md` STEP 1-B の自動選択候補を「直近マイルストーン（アルファベット昇順最小）に一致する Issue のみ」に限定する
- 直近マイルストーンに AI 実行可能 Issue が無い場合の「スキップして終了」動作を明記する
- `docs/dark-factory-workflow.md` §3 の着手順説明を新仕様と整合するよう更新する
- `tests/df-command.test.ts` を新設してTDD で回帰テストを追加する

### やらないこと
- 「直近」の定義をラベル昇順から `due_on` ベースへ変更すること
- `/goal` 側の挙動変更（`/goal` は引数でマイルストーン明示のため対象外）
- 引数で Issue 番号を明示した場合の挙動変更（STEP 1-A は変えない）
- `client` / `server` / `common` のアプリコードへの変更
