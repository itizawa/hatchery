# not-found ユースケース

グローバル 404 画面（未マッチ URL）に関するユースケース。

## 前提条件

- アプリが起動している
- 対象 URL は定義済みルートのいずれにも一致しない（例: `/xyz-nope`）

---

## UC-404-01: 未マッチ URL を開くとカスタム 404 画面が表示される（#529）

**操作**:
1. 存在しない URL（例: `/xyz-nope`）を直接開く。

**期待動作**:
- アプリシェル（ヘッダー・サイドバー）が維持されたまま表示される（レイアウト破綻なし）。
- 「ページが見つかりません」という日本語テキストが表示される（素の英語「Not Found」は表示されない）。
- ホーム（`/`）への導線リンク・ボタンが表示される。

## UC-404-02: 404 画面のホーム導線をクリックするとホームへ遷移できる（#529）

**操作**:
1. 存在しない URL を直接開く。
2. 「ホームへ戻る」ボタン / リンクをクリックする。

**期待動作**:
- ホームフィード（`/`）へ遷移する。
