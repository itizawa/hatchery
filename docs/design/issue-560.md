# Issue #560 設計書: サイドバーの「コミュニティを作る」導線を削除する

## 背景・目的

`client/src/routes/RootLayout.tsx` の `SidebarGlobalNav` コンポーネントに、管理者（`isAdmin(user)`）のみ表示される「コミュニティを作る」ナビゲーションリンク（`/admin?tab=communities`）が存在する。
しかし管理者はすでにサイドバー下部の「管理画面」リンク（`/admin`）から同画面へ遷移できるため、この導線は重複している。
一般ユーザーとの表示差異を取り除き、UIをシンプルにする。

## 受け入れ条件の整理

| # | 条件 | 対応 |
|---|------|------|
| 1 | `SidebarGlobalNav` から「コミュニティを作る」の `ListItem` を削除 | `RootLayout.tsx` L82-98 の `{user && isAdmin(user) && ...}` ブロックを削除 |
| 2 | 「ホーム」「人気」リンクは変わらず表示 | 既存コードは変更しない |
| 3 | 管理者ユーザーにサイドバーに「コミュニティを作る」リンクが表示されない | 削除により達成 |
| 4 | 管理者ユーザーのサイドバー下部「管理画面」リンクは引き続き表示される | 変更しない |
| 5 | `AddIcon` インポートが未使用になる場合は削除 | `AddIcon` はこのブロックのみで使用されているため削除 |
| 6 | `RootLayout.test.tsx` の「コミュニティを作る」関連テスト3ケースを削除 | L412-438 の3ケースを削除 |
| 7 | `pnpm turbo run build test lint` が全て緑 | 実装後に確認 |

## 実装方針

### 変更ファイル

1. **`client/src/routes/RootLayout.tsx`**
   - L82-98: `{user && isAdmin(user) && (<ListItem>...</ListItem>)}` ブロックを削除
   - L13: `import AddIcon from "@mui/icons-material/Add";` を削除
   - JSDocコメント（L42-45）から「コミュニティを作る」の言及を更新

2. **`client/src/routes/RootLayout.test.tsx`**
   - L412-438: 「コミュニティを作る」に関する3テストケースを削除
     - `"admin ユーザーには「コミュニティを作る」が表示され /admin へ導線する"`
     - `"member ユーザーには「コミュニティを作る」が表示されない"`
     - `"未ログインユーザーには「コミュニティを作る」が表示されない"`

### e2e ユースケースの更新

「コミュニティを作る」の削除は管理者ユーザーにとってはユーザー可視の振る舞いが変わるため、`e2e/` を確認して必要であれば更新する。

### TDD 手順

1. テストを更新（3ケース削除）→ 削除前のソースコードがまだ存在するため、テスト実行は成功するはずだが、削除後のテストに対応した実装を確認
2. 実装変更（ソースコード修正）
3. テスト・lint・typecheck を確認

## 考慮事項

- `SidebarGlobalNav` のJSDocコメントにも「コミュニティを作る」への言及があるため更新する
- `isAdmin` インポート自体は `SidebarContent` コンポーネント（管理画面リンク）でも使用しているため、削除しない
- `user` prop も `SidebarGlobalNav` に残るが、将来の拡張性のために維持する（現在は `useLocation` のみで使用する形になる）
  - ただし、`user` prop が使われなくなる場合はコンポーネントの簡略化も検討する

### user prop の扱い

削除後、`SidebarGlobalNav` の `user` prop は不使用になる。
Issue の受け入れ条件には明示されていないが、クリーンなコードにするため prop 自体の削除（または必要なら維持）を検討する。
ただし、Issue の受け入れ条件に含まれていないため、最小限の変更として `user` prop 削除まで実施するかはスコープの解釈次第となる。
受け入れ条件5は「AddIcon のインポートが未使用になる場合は削除」と明記しているため、同様に未使用になる prop・コードは削除する方針で進める。
