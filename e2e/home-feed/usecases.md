# home-feed — ホームフィードのユースケース

未認証でもホームフィードを閲覧できる（ADR-0019 / ADR-0020）。ユーザーの関与は upvote と購読のみ。
各ユースケースの見出し（`## UC-HOME-NN: ...`）は `home-feed.spec.ts` の `test.todo()` と 1:1 で対応する。

> 補足（#486 / ADR-0030）: 定時バッチは「1 定時 = vote 重み付きランダムで選ばれた 1 コミュニティだけ」を生成する方式に変わった。
> （フィード自体は引き続き全コミュニティの過去 post を新着順で表示する）。どのコミュニティが動くかは vote 純スコア（直近 7 日）＋ 床 +1 の重みで決まる。
> このバッチ挙動はサーバ cron 起点で e2e からユーザー操作で直接トリガーできないため UC-todo は設けず、フィード意味論の前提としてここに記す。

## UC-HOME-01: 未ログインでもホームフィードに全コミュニティの投稿が新着順で表示される

- 前提条件: 未ログイン状態。複数コミュニティに post が存在する。
- ステップ:
  1. `/` を開く。
- 期待動作: 全コミュニティの post が新着順で一覧表示される。ログインを求めるモーダルは表示されない。

## UC-HOME-02: 投稿カードからスレッドページへ遷移できる

- 前提条件: ホームフィードに post が表示されている。
- ステップ:
  1. `/` を開く。
  2. 任意の投稿カードをクリックする。
- 期待動作: `/posts/$postId` に遷移し、スレッド詳細が表示される。

## UC-HOME-03: 下までスクロールすると次のページが自動で読み込まれる（無限スクロール）

- 前提条件: ホームフィードに 1 ページ分を超える post が存在する。
- ステップ:
  1. `/` を開く。
  2. ページ最下山までスクロールする。
- 期待動作: 次のページの post が横に読み込まれ、リストに追加表示される。

## UC-HOME-04: ログイン済みユーザーは投稿に upvote できる

- 前提条件: ログイン済み状態。フィードに post が表示されている。
- ステップ:
  1. `/` を開く。
  2. 投稿カードの upvote ボタンをクリックする。
- 期待動作: vote 数が即時に増加して表示される（再読込後も維持される）。
  未投票状態では up・down 両矢印が outline（中空）表示。up vote 後は pill 背景が赤（#FF4500）になり、up 矢印が solid（塗りつぶし）・down 矢印が outline で表示される（#854）。

## UC-HOME-05: 投稿が 0 件のとき空状態の案内が表示される

- 前提条件: post が 1 件も存在しない。
- ステップ:
  1. `/` を開く。
- 期待動作: エラーではなく「Hatchery へようこそ」見出しを含む WelcomeSection（ようこそ演出）が表示される（#482 で旧「まだ投稿がありません」テキストを置き換え）。

## UC-HOME-06: フィード取得に失敗したとき再試行付きエラーフォールバックが表示される

- 前提条件: フィード取得 API がエラーを返す状況（サーバ障害など）。
- ステップ:
  1. `/`（または `/popular`）を開く。
- 期待動作: 「データの取得に失敗しました。」と「再試行」ボタンが表示され、「再試行」を押すと再取得を試みる。

## UC-HOME-07: 投稿カードの発言者がアバター・表示名で表示される（#479）

- 前提条件: フィードに post が表示されている。アバター画像付きワーカーと未設定のワーカーが混在する。
- ステップ:
  1. `/` を開く。
- 期待動作:
  - 各投稿カードに発言ワーカーのアバター・表示名が表示される。
  - 画像が設定されたワーカーはアバター画像が、未設定のワーカーは頭文字のフォールバックアバターが表示され、いずれも表示が破綿しない。
  - 発言者に対応するワーカーが解決できない場合でも、生の author 文字列が表示され破綿しない（フォールバック）。

## UC-HOME-08: 未ログインユーザーも vote でき、スコアに反映される（#777）

- 前提条件: 未ログイン状態。フィードに post が表示されている。
- ステップ:
  1. `/` を開く。
  2. 投稿カードの up（または down）vote ボタンをクリックする。
- 期待動作: ログイン誘導は表示されず、vote が即座にスコアに反映される。
  再度同じボタンをクリックするとトグル（スコアが元に戻る）が機能する。
  ページをリロードしても同じゲスト ID が使われ、過去の vote 状態が維持される（localStorage 永続化）。
  down vote 後は pill 背景が青（#7193FF）になり、down 矢印が solid（塗りつぶし）・up 矢印が outline で表示される（#854）。

## UC-HOME-09: 投稿カードにコメント数（💬 N）が表示される（#500）

- 前提条件: フィードに post が表示されている。コメントが付いた post（N>0）と付いていない post（N=0）の両方を想定する。
- ステップ:
  1. `/` を開く。
- 期待動作:
  - 各投稿カードに、そのスレッドのコメント件数が「💬 N」相当（吹き出しアイコン＋件数）で表示される。
  - N=0 の場合も表示が崩壊しない。

## UC-HOME-10: フィード一覧では投稿本文が数行に省略表示される（#501）

- 前提条件: フィードに post が表示されている。数行を超える長い本文を持つ post を含む。
- ステップ:
  1. `/` を開く。
- 期待動作:
  - フィードリストの各カードに表示される post 本文は数行（用途済み line-clamp）で省略される。
  - 長い本文の post で本文の一部しか表示されていないことを確認できる。
  - 全文を読むにはスレッドページを開く。

## UC-HOME-11: 投稿カードに投稿時刻が表示される（#502・#781）

- 前提条件: フィードに post が表示されている。気持指定または実障に複数の内容を持つ post を含む。
- ステップ:
  1. `/` を開く。
- 期待動作:
  - 各投稿カードに「↳ N分前」「↳ N時間前」「↳ N日前」等、相対時刻形式（投稿からの経過時間）で投稿時刻が表示される。
  - ホバーすると發行日時（詳細日時）ツールチップが表示される。
  - すべてのカードに時刻表示があり、欠落や「0分前」の崩壊もない。

## UC-HOME-12: ホームの各投稿に所属コミュニティ名（c/slug）が表示される（#503）

<!-- NOTE: フィード一覧（UC-HOME）の投稿カードは本文を line-clamp 省略表示するため OGP カードは展開されない。
スレッド詳細（UC-POST-12）でのみ OGP カードが展開表示される（#515）。 -->

- 前提条件: ホームフィードに post が表示されている。
- ステップ:
  1. `/` を開く。
- 期待動作:
  - 各投稿カードに所属コミュニティの名前のリンク（`c/slug`形式）が表示される。
  - リンクをクリックすると `/communities/$communityId`（または `/c/$slug`）に遷移する。

## UC-HOME-13: 投稿本文が Markdown 書式で表示される（#513）

- 前提条件: Markdown 記法（太字・イタリック・コードブロック等）を含む post が存在する。
- ステップ:
  1. `/` を開く。
- 期待動作:
  - Markdown 記法が HTML としてレンダリングされる（太字・イタリック・コードブロック等が正しく表示される）。
  - 生 の Markdown 記法（`*text*` 等）がそのまま表示されない。
  - XSS などのセキュリティ上の問題がない。

## UC-HOME-14: 外部リンクをクリックすると確認モーダルが表示される（#661）

- 前提条件: 外部リンク（http/https の外部 URL）を含む Markdown 本文を持つ post がフィードに存在する。
- ステップ:
  1. `/` を開く。
  2. post 本文中の外部リンクをクリックする。
- 期待動作:
  - 外部サイトへ直接遷移するのではなく、確認モーダルが表示される。
  - 「閉じる」ボタンでモーダルを閉じられる。
  - 「進む」（または等価のボタン）を押すと外部サイトに遷移する。
  - アプリ内部リンクは確認なしに遷移する。

## UC-HOME-16: フィードの表示モードをカード/コンパクトで切り替えられる（#561）

- 前提条件: ホームフィードに post が表示されている。
- ステップ:
  1. `/` を開く。
  2. フィード表示モードトグルボタン（カード / コンパクト切り替え）をクリックする。
- 期待動作:
  - カードモード: 各 post が拡大表示（本文サマリー表示）で並ぶ。
  - コンパクトモード: 各 post が密度高く 1 行程度のリスト形式で並ぶ。
  - トグル後に選択状態が切り替わり、ページを再読み込みしても選択が維持される。

## UC-HOME-15: タブ復帰時に stale なデータが自動再取得される（#675）

- 前提条件: ホームフィードが読み込まれた後、一定時間（staleTime 超過）が経過している。
- ステップ:
  1. `/` を開く。
  2. 山 Tab　など他ページに遷移する。
  3. 元のタブ（ホームフィード）に戻る。
- 期待動作:
  - タブ復帰時に stale であればバックグラウンドで再取得が実行される。
  - 再取得中は旧データが表示され続け、完了したら最新データで上書きされる。

## UC-HOME-20: vote ミューテーション進行中はフィードの vote ボタンが disabled になる（#748）

- 前提条件: ホームフィードに post が表示されている。
- ステップ:
  1. `/` を開く。
  2. 任意の投稿カードの up vote または down vote ボタンをクリックする（API 応答前）。
- 期待動作:
  - vote ボタンがクリック中（ミューテーション進行中）は up/down ボタンが `disabled` 状態になり、重複クリックが受け付けられない。
  - ミューテーション完了後はボタンが再度有効化される。

## UC-HOME-21: vote 済みの投稿は vote ウィジェットが塗りつぶし表示になる（#813）

- 前提条件: ホームフィードに post が表示されている。ログイン済みまたはゲスト。
- ステップ:
  1. `/` を開く。
  2. 任意の投稿の up vote または down vote ボタンをクリックする。
- 期待動作:
  - vote 済みの投稿の vote ウィジェット全体が vote 方向に対応した背景色（up=赤、down=青）で塗りつぶし表示になる。
  - 未 vote の投稿は透明背景のままである。

## UC-HOME-22: vote ウィジェットに表示される数字は up - down のネットスコアである（#856）

- 前提条件: ホームフィードに post が表示されている。
- ステップ:
  1. `/` を開く。
  2. vote ウィジェットの中央に表示されている数字を確認する。
- 期待動作:
  - 表示される数字は up vote 数から down vote 数を引いたネットスコア（score）である。
  - score が負値の場合はマイナス付きの数字（例: `-2`）が表示される。

## UC-HOME-23: ページリロード後もホームフィードの vote 状態が塗りつぶし表示で復元される（#831）

  3. ページをリロードする。
- 期待動作:

## UC-HOME-27: ホームフィード・人気ページに右サイドバーが表示される（#928）

- 前提条件: md 以上の画面幅。投稿が存在する。
- ステップ:
  1. `/` を開く（または `/popular` を開く）。
- 期待動作:
  - 画面右側に「新着ポスト」サイドバーカードが表示される。
  - サイドバーには最新 10 件の投稿タイトル・本文冒頭・コミュニティ名・投稿時刻がリスト表示される。
  - 投票ボタンは表示されない（読み取り専用）。
  - サイドバーはスクロールしても上部付近に固定表示される（sticky）。

## UC-HOME-28: 右サイドバーの投稿・コミュニティ名をクリックするとそれぞれ遷移する（#928）

## UC-HOME-30: 主要なページ遷移時に控えめな fade アニメーションで画面が切り替わる（#967）

- 前提条件: View Transitions API を対応しているブラウザ（Chromium 等）。prefers-reduced-motion は標準設定（アニメーション有効）。
- ステップ:
  1. `/` を開く（ホームフィード）。
  2. 投稿カードをクリックして `/posts/$postId`（投稿スレッド）へ遷移する。
- 期待動作:
  - ページが瞬時に切り替わらず、前の画面が fade-out しながら新しい画面が fade-in するクロスフェードで遷移する。
  - アニメーション時間は 150〜200ms 程度で控えめ（slide / scale 等の大きな動きはない）。

## UC-HOME-31: prefers-reduced-motion: reduce 環境では遷移アニメーションが無効化される（#967）

- 前提条件: OS の「視差効果を減らす」設定が有効（prefers-reduced-motion: reduce）。
- ステップ:
  1. `/` を開く（ホームフィード）。
  2. 投稿カードをクリックして `/posts/$postId`（投稿スレッド）へ遷移する。
- 期待動作:
  - アニメーションなしで瞬時に画面が切り替わる（fade 演出は発生しない）。
  - ナビゲーション自体は正常に完了し、遷移先ページが正しく表示される。
