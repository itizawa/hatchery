# 設計書: CLAUDE.md に Anthropic Cookbook の frontend aesthetics プロンプトを埋め込む (#795)

## 1. 目的 / 背景

Anthropic 公式の "Prompting for frontend aesthetics" Cookbook には、AI に UI を生成させる際の aesthetics プロンプト戦略が公開されている。このプロジェクト向けに適用した内容を CLAUDE.md に固定ブロックとして追記することで、AI がフロントエンドコードを書くたびにデザイン品質の指針に自動で従うようにする。

現在の CLAUDE.md には `## デザインシステム（#792）` が存在し「何を使うか・何を禁止するか」の設計方針は定まっているが、「どのようにプロンプトするか」という生成時の指示手法は未整備（line 161 に「aesthetics プロンプト文面は Issue #783 で別途追加予定」と注記あり）。

## 2. スコープ（やること / やらないこと）

**やること:**
- `CLAUDE.md` に `## Frontend Aesthetics（UI 生成時の指針）` セクションを追加
- 3 戦略（デザイン次元の個別指定 / 参照先の名指し / Generic デフォルトの明示禁止）を記載
- `## デザインシステム（#792）` 冒頭の「#783 で別途追加予定」注記を削除（本 Issue で実装済みのため）

**やらないこと:**
- aesthetics プロンプトを API コール時に自動付与する仕組み（別途検討）
- フォントの実インストール・適用
- デザインシステム（#792）既存内容の変更

## 3. 受け入れ条件（テストに落とせる粒度で箇条書き）

1. `CLAUDE.md` に `## Frontend Aesthetics（UI 生成時の指針）` セクションが存在する
2. 同セクションに以下の 3 戦略が記載されている
   - **デザイン次元の個別指定**: タイポグラフィ・カラー・モーション・背景を分けて指示する方針
   - **参照先の名指し**: "Linear.app 風" / "Vercel Dashboard 風" など具体的な参照元を挙げる方針
   - **Generic なデフォルトの明示禁止**: AI がデフォルトしがちなパターン（白紫グラデーション・Inter フォント等）を禁止リストとして記載
3. 追記内容が `## デザインシステム（#792）` と矛盾しない（役割分担: #792 は設計方針、本節は生成指針）
4. `## デザインシステム（#792）` 冒頭の「#783 で別途追加予定」注記が削除されている
5. `pnpm turbo run build lint test` が通る（CLAUDE.md はコードではないため lint/test への影響なし）

## 4. 設計方針（アーキ・データ構造・主要モジュール）

CLAUDE.md の末尾（`## デザインシステム（#792）` の直後）に新セクションを追加する。
`## デザインシステム（#792）` が「ルール集（何を使う・何を禁止）」、`## Frontend Aesthetics（UI 生成時の指針）` が「プロンプト手法（どのように指示する）」という役割分担を冒頭に明示する。

3 戦略の構成:
- **戦略 1**: デザイン次元を分けて指定する表（次元 × 記述例）
- **戦略 2**: このプロジェクトの承認済み参照先リスト
- **戦略 3**: Generic デフォルト禁止リスト（デザインシステムの禁止パターンとリンク）

## 5. 影響範囲 / 既存への変更

- **対象ファイル**: `CLAUDE.md`（プロジェクトルート）のみ
- **既存コードへの影響**: なし（CLAUDE.md はコードではない）
- **既存セクションへの変更**: `## デザインシステム（#792）` 冒頭の 1 文を削除

## 6. テスト計画（TDD で書くテスト一覧）

CLAUDE.md はコードではなく、直接テストで検証するファイルではない。ただし `tests/` にリポジトリ規約テスト（`pnpm test:repo`）が存在する場合は実行して緑を確認する。

代替検証:
- `git diff HEAD -- CLAUDE.md` で変更差分を目視確認し受け入れ条件 1〜4 を満たすことを確認
- `pnpm turbo run build lint test`（またはローカルで `pnpm test:repo`）で緑を確認

## 7. リスク・未決事項

- `## デザインシステム（#792）` との内容重複: 禁止パターンリストが両セクションに登場するが、デザインシステムは「何を禁止するか（ルール）」、本節は「プロンプト時にも再掲して AI に確実に適用させる（手法）」という位置付けで意図的な重複。
- Issue #783 との関係: Issue #783 は本 Issue #795 を指す表記ゆれの可能性あり。#783 が別途存在する場合でも、本 Issue で `## Frontend Aesthetics` セクションを追加すれば受け入れ条件は満たされる。
