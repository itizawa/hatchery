# Issue #608 設計書: ポスト生成プロンプトで特定ユーザーへの直接呼びかけを抑制する

## 背景・目的

`server/src/batch/buildCommunityPrompt.ts` の `TONE_GUIDELINES` は「さん付けを避けフランクに」と指示しているが、
日本のネットコミュニティ（5ch・個人開発界隈 Reddit 的）では、そもそも特定ユーザーを名指しで呼びかけること自体が少ない。
現状の生成では「○○、」「○○さ、」のような冒頭呼称が出てしまい、自然な掲示板文体と乖離している。

プロンプト側の指示改善のみで、名指し呼びかけを抑制する（後処理フィルタはスコープ外）。

## 受け入れ条件 → 対応

1. `TONE_GUIDELINES` に「コメント内で他ワーカーを名指し呼びかけすることを避ける（「○○、」「○○さ、」等の冒頭呼称は使わない）」旨のルールを追記。
   - `呼称:` の行に「名指しの直接呼びかけ（冒頭で相手の名前を呼ぶ「○○、」「○○さ、」等）はしない」旨を追加する。
2. 自己監査セクションに「コメント本文が特定ユーザーへの直接呼びかけで始まっていないか」のチェック項目を追記。
3. `buildCommunityPrompt.test.ts` に、生成プロンプト文字列／`TONE_GUIDELINES` が「名指し呼びかけを避けるルール」を含むことを検証するテストを追加。
4. 既存テスト（`buildCommunityPrompt.test.ts`・`aiMessageGenerator.test.ts`）がすべて通過。
5. `pnpm turbo run build test lint` が緑。

## 設計判断の要点

- 変更はプロンプト文字列定数のみ。ドメインロジック・型・スキーマは変更しない。
- `TONE_GUIDELINES` は `## 487` 由来の「脚本ルール層（共通エンジン部）」であり全 community 共通注入されるため、ここにルールを足せば全コミュニティで効く。
- 自己監査セクション（`buildCommunityPrompt` 末尾）にチェック項目を1行追加する。
- 検証キーワードは表現に依存しないよう「名指し」「呼びかけ」を軸にテストする（既存テストが「さん付け」等のキーワードで検証している方針に合わせる）。

## スコープ外

- 生成結果に対する後処理フィルタリング（呼びかけ除去）は今回スコープ外。
