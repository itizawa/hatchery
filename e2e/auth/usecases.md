# auth — ログイン・ログアウトのユースケース

対応画面: `client/src/routes/LoginScene.tsx`（/login）・`client/src/components/AppHeader.tsx`（ログアウト）

各ユースケースの見出し（`## UC-AUTH-NN: ...`）は `auth.spec.ts` の `test.todo()` と 1:1 で対応する。

## UC-AUTH-01: ログイン画面の表示

- 前提条件: 未ログイン状態。
- ステップ:
  1. `/login` を開く。
- 期待動作: 「ログイン」見出し・ID 入力欄・パスワード入力欄・送信ボタンが表示される。

## UC-AUTH-02: 正しい認証情報でログインに成功しホームへ遷移する

- 前提条件: 有効なユーザー（loginId / password）が存在する。未ログイン状態。
- ステップ:
  1. `/login` を開く。
  2. 有効な ID とパスワードを入力して送信する。
- 期待動作: ホーム（`/`）へリダイレクトされ、ヘッダーにログイン済みユーザーのメニューが表示される。

## UC-AUTH-03: 誤った認証情報ではエラーメッセージが表示される

- 前提条件: 未ログイン状態。
- ステップ:
  1. `/login` を開く。
  2. 存在しない ID または誤ったパスワードを入力して送信する。
- 期待動作: 「ID またはパスワードが正しくありません」が表示され、`/login` に留まる。

## UC-AUTH-04: 未入力で送信するとフィールド必須エラーが表示される

- 前提条件: 未ログイン状態。
- ステップ:
  1. `/login` を開く。
  2. ID・パスワードを空のまま送信する。
- 期待動作: 「ID は必須です」「パスワードは必須です」のバリデーションエラーが表示され、API は呼ばれない。

## UC-AUTH-05: ログアウトすると未ログイン状態に戻る

- 前提条件: ログイン済み状態。
- ステップ:
  1. ヘッダーのアカウントメニューを開く。
  2. ログアウトを実行する。
- 期待動作: 未ログイン状態になり、ヘッダーがゲスト表示（ログイン導線）に切り替わる。

## UC-AUTH-06: 未ログインで認証必須ページ（/account）にアクセスすると /login へリダイレクトされる

- 前提条件: 未ログイン状態。
- ステップ:
  1. `/account` を直接開く。
- 期待動作: `/login` へリダイレクトされる。
