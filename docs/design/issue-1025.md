## 3. 受け入れ条件（テストに落とせる粒度）

1. `weight` に負値（例: `-5`）を含むコミュニティ配列を渡し、負の重みが 0 として扱われクラッシュしないこと・選定結果が妥当であることを検証するテストを追加する。
2. 浮動小数点誤差フォールバックループに到達するケース（`rng: () => 1.0` で `r === total` を模擬）で、最後の正の重みを持つコミュニティが返されることを検証するテストを追加する。
3. 追加後、`selectWeightedCommunity.ts` の branches カバレッジが現状（88.88%）を上回ること。
4. `pnpm turbo run build|test|lint` が緑であること。

## 4. 設計方針

### 負の重みテスト
- `weight: -5` を含む配列を渡す。
- `Math.max(0, -5) = 0` のため、負重みコミュニティは選ばれない。
- 正の重みを持つコミュニティだけが選ばれることを検証。

### 浮動小数点誤差フォールバックテスト
- `rng: () => 1.0` を注入（`rng() * total = total`）。
- メインループ: `r < cumulative` は最後のコミュニティ（`cumulative === total`, `r === total`）で false → 全コミュニティを通過。
- フォールバック逆順ループが起動し、最後の正の重みを持つコミュニティを返す。
- 末尾に重み 0 のコミュニティを置いたケースも追加し、逆順ループが正しく最後の「正の重み」コミュニティを見つけることを確認。
