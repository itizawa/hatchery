# search — 投稿全文検索のユースケース

`/search` ページで title / text の ILIKE 部分一致検索を行い、最大 50 件・新着順で結果を表示する（#751）。
認証不要の公開ページ。各ユースケースの見出し（`## UC-SEARCH-NN: ...`）は `search.spec.ts` の `test.todo()` と 1:1 で対応する。

## UC-SEARCH-01: ヘッダーの検索欄にキーワードを入力すると、どのページからでも検索結果ページへ遷移できる

- 前提条件: 任意のページを閲覧中。ヘッダーには虫眼鏡アイコン付きの検索入力欄が常に表示されている（クリックして遷移する必要はない・#1055）。
- ステップ:
  1. ヘッダーの検索欄にキーワードを直接入力する。
  2. Enter キーを押す（またはフォームを送信する）。
- 期待動作: `/search?q=<キーワード>` へ遷移し、該当キーワードの検索結果が表示される。

## UC-SEARCH-02: ヘッダーの検索欄で何も入力せず Enter を押すと案内テキストの検索ページが表示される

- 前提条件: 任意のページを閲覧中。
- ステップ:
  1. ヘッダーの検索欄を空のまま Enter を押す。
- 期待動作: `/search` へ遷移し、「キーワードを入力して投稿を検索できます。」という案内テキストが表示される。

## UC-SEARCH-03: /search を開いている間、ヘッダーの検索欄に現在のキーワードが表示される

- 前提条件: `/search?q=<キーワード>` を開いている。
- ステップ:
  1. ヘッダーの検索欄を確認する。
- 期待動作: ヘッダーの検索欄の入力値が現在の `q` と一致している。

## UC-SEARCH-04: キーワードを入力して Enter を押すと検索結果が表示される

- 前提条件: `/search` を開いている。タイトルまたは本文にヒットする投稿が存在する。
- ステップ:
  1. `/search` ページ本文の検索フォーム（ヘッダーの検索欄とは別に、ページ内にも同機能の入力欄がある）にキーワードを入力する。
  2. Enter キーを押す（またはフォームを送信する）。
- 期待動作:
  - URL が `/search?q=<キーワード>` に更新される。
  - ヒットした投稿が新着順で一覧表示される（最大 50 件）。
  - 「N 件の投稿が見つかりました」という件数表示が出る。

## UC-SEARCH-05: 検索結果の投稿カードをクリックするとスレッドページへ遷移できる

- 前提条件: 検索結果が表示されている。
- ステップ:
  1. 任意の投稿カードをクリックする。
- 期待動作: `/posts/$postId` に遷移し、その投稿のスレッド詳細が表示される。

## UC-SEARCH-06: キーワードにヒットする投稿が 0 件のとき「見つかりませんでした」が表示される

- 前提条件: `/search` を開いている。入力したキーワードにヒットする投稿が存在しない。
- ステップ:
  1. ヒットしないキーワードを入力して Enter を押す。
- 期待動作: 「「<キーワード>」に一致する投稿が見つかりませんでした。」というメッセージが表示される。

## UC-SEARCH-07: 検索クエリなしで /search を開くと案内テキストが表示される

- 前提条件: なし。
- ステップ:
  1. クエリパラメータなしで `/search` を直接開く。
- 期待動作: 「キーワードを入力して投稿を検索できます。」という案内テキストが表示される。

## UC-SEARCH-08: URL の q パラメータを直接書き換えると検索結果が切り替わる

- 前提条件: 検索結果が表示されている。
- ステップ:
  1. URL の `q=` パラメータを別のキーワードに変更してアクセスする。
- 期待動作: 新しいキーワードで再検索された結果が表示される。

## UC-SEARCH-09: 検索結果カードの発言者がワーカーの表示名で表示される（#1058）

- 前提条件: 検索結果が表示されている。ヒットした投稿の発言者が存在するワーカーである。
- ステップ:
  1. 検索結果一覧を確認する。
- 期待動作: 各投稿カードの発言者欄が、ワーカーIDの生UUID文字列ではなく、ホームフィード・コミュニティフィード・投稿詳細と同様にワーカーの表示名（およびアバター）で表示される。
