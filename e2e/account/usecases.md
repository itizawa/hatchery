# account — アカウント設定（プロフィール編集）のユースケース

対応画面: `client/src/routes/AccountScene.tsx`（/account）・対応 API: `client/src/api/auth.ts`（`updateProfile` / PATCH `/auth/me`）

/account は認証必須（未ログインはホーム上にログインモーダル・auth/usecases.md UC-AUTH-05 を参照）。
各ユースケースの見出し（`## UC-ACCOUNT-NN: ...`）は `account.spec.ts` の `test.todo()` と 1:1 で対応する。

## UC-ACCOUNT-01: 表示名・プロフィール画像 URL を変更して保存できる

- 前提条件: ログイン済みで `/account` を開いている。
- ステップ:
  1. 表示名を変更する。
  2. 「保存」ボタンをクリックする。
- 期待動作: 「保存しました」の成功 Snackbar が表示され、変更内容がフォームに反映される。

## UC-ACCOUNT-02: 変更が無いとき保存ボタンが無効化される

- 前提条件: ログイン済みで `/account` を開いている。
- ステップ:
  1. フォームを編集せずに保存ボタンの状態を確認する。
- 期待動作: 「保存」ボタンが無効（disabled）。表示名を初期値から変更すると有効になり、初期値へ戻すと再び無効になる。

## UC-ACCOUNT-03: 不正な URL を入力すると保存できずエラーが表示される

- 前提条件: ログイン済みで `/account` を開いている。
- ステップ:
  1. プロフィール画像 URL に URL として不正な値を入力してフォーカスを外す。
- 期待動作: 「有効な URL を入力してください」等のバリデーションエラーが表示され、保存できない。

## UC-ACCOUNT-04: プロフィール更新に失敗するとエラー内容が表示される（#472）

- 前提条件: ログイン済みで `/account` を開いている。サーバ側の更新がエラーを返す状況（バリデーション失敗・権限切れ・サーバエラー・ネットワーク断など）。
- ステップ:
  1. 表示名を変更する。
  2. 「保存」ボタンをクリックし、更新がエラーになる。
- 期待動作: エラー Snackbar が表示され、更新に失敗した旨（可能ならサーバが返したエラー内容）が示される。成功 Snackbar（「保存しました」）は表示されず、ユーザーが保存成否を取り違えない。

## UC-ACCOUNT-05: 初回ログイン直後（?welcome=1）に表示名設定を促す歓迎メッセージが表示される

- 前提条件: 初回ログイン完了後、`/account?welcome=1` へ遷移している（auth/usecases.md UC-AUTH-10 を参照）。
- ステップ:
  1. `/account?welcome=1` を開く。
- 期待動作: アカウント設定フォームの上部に、表示名の設定を促す歓迎メッセージ（「ようこそ Hatchery へ！…」）が表示される。`?welcome=1` が付かない通常の `/account` 表示では歓迎メッセージは表示されない。

## UC-ACCOUNT-06: VAPID 公開鍵が未設定の場合、プッシュ通知ボタンが非表示になる（#798）

- 前提条件: `VITE_VAPID_PUBLIC_KEY` 環境変数が未設定の状態でログイン済みで `/account` を開いている。
- ステップ:
  1. `/account` を開く。
- 期待動作: プッシュ通知セクション（「プッシュ通知」ラベルとボタン）が表示されない。

## UC-ACCOUNT-07: プッシュ通知を購読できる（#798）

- 前提条件: `VITE_VAPID_PUBLIC_KEY` が設定済み。ログイン済みで `/account` を開いている。ブラウザが Web Push に対応しており、通知権限が未設定（または許可済み）。
- ステップ:
  1. `/account` を開く。
  2. 「新着通知を受け取る」ボタンをクリックする。
  3. ブラウザの通知許可ダイアログで「許可」を選択する。
- 期待動作: ボタンが「通知をオフにする」に切り替わり、以後の定時バッチ実行後にプッシュ通知が届く。

## UC-ACCOUNT-08: プッシュ通知を解除できる（#798）

- 前提条件: `VITE_VAPID_PUBLIC_KEY` が設定済み。ログイン済みで `/account` を開いており、すでに購読中。
- ステップ:
  1. `/account` を開く。
  2. 「通知をオフにする」ボタンをクリックする。
- 期待動作: ボタンが「新着通知を受け取る」に戻り、以後のプッシュ通知が届かなくなる。

## UC-ACCOUNT-09: 通知権限を拒否するとブラウザ設定への誘導メッセージが表示される（#798）

- 前提条件: `VITE_VAPID_PUBLIC_KEY` が設定済み。ログイン済みで `/account` を開いており、ブラウザの通知権限が「拒否」になっている。
- ステップ:
  1. `/account` を開く。
- 期待動作: 「通知がブロックされています。ブラウザの設定から通知を許可してください。」というメッセージが表示され、購読ボタンは表示されない。
