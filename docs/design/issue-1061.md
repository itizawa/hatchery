# 設計書: concept.md の「down vote は持たない」という記述が ADR-0025 の down vote 導入と矛盾している (#1061)

## 1. 目的 / 背景

`concept.md`（「ユーザーの関与モデル」章）は「vote は up のみ・down vote は持たない」と記述しているが、`docs/adr/0025-down-vote-reddit-action-bar.md` で up/down 両方向の vote（Reddit 風アクションバー）が正式決定され、本番 UI にも実装済み。正本間の矛盾が今後の実装判断（vote に触れる新機能の前提理解）を誤らせるリスクがあるため解消する。

## 2. スコープ（やること / やらないこと）

- やること: `concept.md` 内の down vote に関する記述を ADR-0025 の内容に合わせて更新する。冒頭のステータス注記に ADR-0025 を追記する。
- やらないこと: コード変更・テスト追加（ドキュメントのみの修正で挙動変更なし）。他の ADR との整合チェックは本 Issue のスコープ外。

## 3. 受け入れ条件（テストに落とせる粒度で箇条書き）

1. `concept.md`「ユーザーの関与モデル」章の「vote は up のみ」「down vote は持たない」という記述が、ADR-0025 の内容（up/down 両方向・score はネット値・down 累積数は非公開）に整合する記述に置き換わっていること。
2. 同章内の「並びは vote と新着で制御する」等、down vote 導入で意味が変わりうる近傍の記述に矛盾がないこと。
3. `concept.md` 冒頭のステータス注記（3行目）の参照 ADR 一覧に ADR-0025 が追記されていること。
4. コード変更を伴わないため `pnpm turbo run build test lint` の実行は不要（PR にその旨を明記する）。

## 4. 設計方針（アーキ・データ構造・主要モジュール）

Markdown ドキュメントのテキスト修正のみ。ADR-0025 の決定事項（`direction` enum・score はネット値・down 累積数非公開・Reddit 風アクションバー）を要約して反映する。

## 5. 影響範囲 / 既存への変更（対象ワークスペース: client / server / common / docs）

対象は `docs/`（`concept.md`）のみ。client / server / common への変更なし。

## 6. テスト計画（TDDで書くテスト一覧）

ドキュメントのみの修正のためテスト対象コードなし。TDD サイクルは適用しない（受け入れ条件 4 参照）。

## 7. リスク・未決事項

なし。
