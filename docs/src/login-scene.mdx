# 画面設計書: ログイン（LoginDialog）

> 関連: Issue #26 / Issue #70 / Issue #454 / Issue #455
> ADR-0003（client）・ADR-0007（docs）

## 目的

ログインは MUI Dialog（モーダル）で表示する（Issue #454）。
`/login` へのアクセスは `/?login=1` へリダイレクトし、ホームフィードを背景に保ちつつモーダルを開く。
認証手段は **Google OAuth のみ**（Issue #455）。

## URL / ルート

| URL | 動作 | 認証 |
|-----|------|------|
| `/login` | `/?login=1` へリダイレクト | 不要 |
| `/?login=1` | ホームフィード上にログインモーダルを表示 | 不要 |

全画面の URL（ルート）一覧は Storybook の「設計/画面URL一覧（ルーティング表）」ページ（正本: `docs/design/url-list.md`）を参照してください。

## 認証要否

認証不要。モーダルはホームフィード（公開ページ）の上に重ねて表示する。

## 状態一覧

| 状態 | 説明 | ストーリー |
|------|------|----------|
| 開いた状態 | ログインモーダルが表示されている | `Open` |
| 閉じた状態 | モーダル非表示（何も表示されない） | `Closed` |

## データ契約

- Google OAuth リダイレクト → コールバック後に `GET /auth/me` で認証状態を確認
- ログイン成功後は `/?login=1` search param を除去してホームへ遷移
