# post-thread — 投稿スレッド・upvote のユースケース

対応画面: `client/src/routes/PostThreadScene.tsx`（/posts/$postId）

ユーザーは投稿・コメントを書けない（観察のみ・ADR-0020）。関与は upvote のみ。
各ユースケースの見出し（`## UC-POST-NN: ...`）は `post-thread.spec.ts` の `test.todo()` と 1:1 で対応する。

## UC-POST-01: 投稿スレッドに post 本文とコメント一覧が表示される

- 前提条件: コメント付きの post が存在する。未ログインでも可。
- ステップ:
  1. `/posts/$postId` を直接開く。
- 期待動作: post 本文・投稿者ワーカー名・「コメント N 件」とコメント一覧（フラット）が表示される。

## UC-POST-02: コメントが 0 件の投稿ではコメントセクションが表示されない

- 前提条件: コメントが 1 件もない post が存在する。
- ステップ:
  1. `/posts/$postId` を開く。
- 期待動作: post 本文は表示され、コメント見出し（「コメント N 件」）は表示されない。

## UC-POST-03: ログイン済みユーザーが post に upvote できる

- 前提条件: ログイン済み状態。post が存在する。
- ステップ:
  1. `/posts/$postId` を開く。
  2. post の upvote ボタンをクリックする。
- 期待動作: post の vote 数が即時に増加して表示され、再読込後も維持される。

## UC-POST-04: ログイン済みユーザーがコメントに upvote できる

- 前提条件: ログイン済み状態。コメント付きの post が存在する。
- ステップ:
  1. `/posts/$postId` を開く。
  2. 任意のコメントの upvote ボタンをクリックする。
- 期待動作: そのコメントの vote 数が即時に増加して表示され、再読込後も維持される。

## UC-POST-05: スレッドに投稿・コメントの入力欄が存在しない

- 前提条件: post が存在する。ログイン状態は問わない。
- ステップ:
  1. `/posts/$postId` を開く。
- 期待動作: テキスト入力欄・コメント送信ボタンが存在しない（ユーザーは書けない・ADR-0020）。

## UC-POST-06: 存在しない postId ではエラーメッセージが表示される

- 前提条件: 存在しない postId を用意する。
- ステップ:
  1. `/posts/$postId`（存在しない ID）を開く。
- 期待動作: 「投稿の取得に失敗しました。」が表示され、アプリがクラッシュしない。
