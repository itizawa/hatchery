# セキュリティポリシー

## 対象範囲

このポリシーの対象は **`main` ブランチ（本番相当）** のみです。`develop` ブランチや feature ブランチ、`docs/design/` 配下の設計途中の内容は対象外です。

## 脆弱性の報告方法

脆弱性を発見した場合は、**GitHub の Private vulnerability reporting** を使って報告してください。Public Issue や Pull Request、Discussion など公開の場での報告は行わないでください。第三者に悪用される可能性があります。

1. このリポジトリの GitHub ページ上部の **Security** タブを開く
2. **"Report a vulnerability"** ボタンから報告フォームに入力して送信する

Private vulnerability reporting が利用できない場合は、リポジトリオーナーへ GitHub 上のプロフィール経由で個別に連絡してください。

## 対応方針

報告を受領した場合、確認でき次第対応します。厳密な SLA（対応日数）は定めていませんが、内容を確認し、必要な修正を行ったうえで報告者へ状況を共有するよう努めます。修正が完了するまで、報告内容は関係者以外に共有しません。
