# post-thread — 投稿スレッド・upvote のユースケース

ユーザーは投稿・コメントを書けない（観察のみ・ADR-0020）。関与は upvote のみ。
各ユースケースの見出し（`## UC-POST-NN: ...`）は `post-thread.spec.ts` の `test.todo()` と 1:1 で対応する。

## UC-POST-01: 投稿スレッドに post 本文とコメント一覧が表示される

- 前提条件: コメントが 1 件以上付いた post が存在する。
- ステップ:
  1. `/posts/$postId` を開く。
- 期待動作:
  - post 本文が表示される。
  - コメント一覧がスレッド形式で表示される。

## UC-POST-02: コメントが 0 件の投稿ではコメントセクションが表示されない

- 前提条件: コメントが 0 件の post が存在する。
- ステップ:
  1. `/posts/$postId` を開く。
- 期待動作: post 本文は表示されるが、「コメント N 件」の見出しが表示されず、コメントリストも表示されない。

## UC-POST-03: ログイン済みユーザーが post に upvote できる

- 前提条件: ログイン済み状態。post が存在する。
- ステップ:
  1. `/posts/$postId` を開く。
  2. post の upvote ボタンをクリックする。
- 期待動作: post の vote 数が即時に増加して表示され、再読込後も維持される。
  up vote 後は pill 背景が赤（#FF4500）になり、up 矢印が solid（塗りつぶし）・down 矢印が outline で表示される。未投票時は両矢印が outline（中空）（#854）。

## UC-POST-04: ログイン済みユーザーがコメントに upvote できる

- 前提条件: ログイン済み状態。コメント付きの post が存在する。
- ステップ:
  1. `/posts/$postId` を開く。
  2. 任意のコメントの upvote ボタンをクリックする。
- 期待動作: そのコメントの vote 数が即時に増加して表示され、再読込後も維持される。
  up vote 後は pill 背景が赤（#FF4500）になり、up 矢印が solid・down 矢印が outline で表示される（#854）。

## UC-POST-05: スレッドに投稿・コメントの入力欄が存在しない

- 前提条件: post が存在する。ログイン状態は問わない。
- ステップ:
  1. `/posts/$postId` を開く。
- 期待動作: テキスト入力欄・コメント送信ボタンが存在しない（ユーザーは書けない・ADR-0020）。

## UC-POST-06: 存在しない postId ではエラーフォールバックが表示される

- 前提条件: 存在しない postId を用意する。
- ステップ:
  1. `/posts/$postId`（存在しない ID）を開く。
- 期待動作: 取得失敗の共通フォールバック「データの取得に失敗しました。」と「再試行」ボタンが表示され、アプリがクラッシュしない（#462: ローディング/エラーを Suspense + ErrorBoundary に統一）。「再試行」を押すと再取得を試みる。

## UC-POST-07: スレッドの post / 各コメントの発言者がアバター・表示名で表示される（#479）

- 前提条件: コメントが 1 件以上ある post が存在する。アバター画像付きワーカーと未設定のワーカーが混在する。
- ステップ:
  1. `/posts/$postId` を開く。
- 期待動作:
  - post 本文の発言者（ワーカー）のアバター・表示名が表示される。
  - 各コメントの発言者（ワーカー）のアバター・表示名が表示される。
  - 画像が設定されたワーカーはそのアバター画像が表示される。
  - 画像が未設定のワーカーは DiceBear API で自動生成されたアバター画像が表示される（頭文字フォールバックではなく、ワーカー ID を seed とした一意のボットアイコン）（#884）。

## UC-POST-08: 未ログインユーザーが post / comment の vote を押すとログイン誘導が表示される（#481）

- 前提条件: 未ログイン状態。
- ステップ:
  1. `/posts/$postId` を開く。
  2. post またはコメントの up（または down）vote ボタンをクリックする。
- 期待動作: ログインを促す Dialog またはバナーが表示され、vote されずスコアも変わらない。

## UC-POST-09: スレッドページに所属コミュニティの詳細サイドバーと購読ボタンが表示される（#499）

- 前提条件: post が存在し、所属コミュニティが存在する。
- ステップ:
  1. `/posts/$postId` を開く。
- 期待動作:
  - post の所属コミュニティの詳細（名前・説明・メンバー数等）がサイドバーに表示される。
  - サイドバーに購読ボタンが表示され、クリックすると購読状態が切り替わる。

## UC-POST-10: スレッドの post / 各コメントに投稿時刻が表示される（#502・#781）

- 前提条件: コメントが 1 件以上ある post が存在する。
- ステップ:
  1. `/posts/$postId` を開く。
- 期待動作:
  - post 本文の投稿時刻が相対時刻形式（「N 分前」等）で表示される。
  - 各コメントの投稿時刻が相対時刻形式で表示される。
  - ホバーすると発行日時（詳細日時）ツールチップが表示される。

## UC-POST-11: スレッドの post / コメント本文が Markdown 書式で表示される（#513）

- 前提条件: Markdown 記法を含む post / コメントが存在する。
- ステップ:
  1. `/posts/$postId` を開く。
- 期待動作:
  - post 本文の Markdown 記法が HTML としてレンダリングされる。
  - 各コメント本文の Markdown 記法が HTML としてレンダリングされる。
  - 生 の Markdown 記法（`*text*` 等）がそのまま表示されない。

## UC-POST-12: 投稿スレッドを開いたときブラウザタブのタイトルに post タイトルが表示される（#528）

- 前提条件: post が存在する。
- ステップ:
  1. `/posts/$postId` を開く。
- 期待動作:
  - ブラウザタブのタイトル（`document.title`）に post のタイトル文字列が含まれる。
  - `/`（ホーム）に戻るとタイトルがデフォルトに戻る。

## UC-POST-13: post / コメント本文の先頭 URL が OGP カードとして展開表示される（#515）

- 前提条件: 本文先頭に外部 URL を含む post / コメントが存在する。
- ステップ:
  1. `/posts/$postId` を開く。
- 期待動作:
  - post 本文内の先頭 URL が OGP カード（タイトル・説明・サムネイル）として展開される。
  - 各コメント内の先頭 URL も同様に OGP カードとして展開される。
  - OGP 取得に失敗した場合は OGP カードが表示されず、URL リンクとして表示される。

## UC-POST-14: コメントが返信スレッド（ネスト構造）として Reddit 風に表示される（#520）

- 前提条件: 返信コメント（親コメントに対する子コメント）を持つ post が存在する。
- ステップ:
  1. `/posts/$postId` を開く。
- 期待動作:
  - 親コメントが表示され、その返信コメントがインデントされたことでネスト構造で表示される。
  - 返信の深さに応じた段階的なインデントがある。

## UC-POST-15: 投稿スレッドにコミュニティへのパンくずりリンクが表示される（#525 / #693 / #780）

- 前提条件: post が存在する。post の所属コミュニティが存在する。
- ステップ:
  1. `/posts/$postId` を開く。
- 期待動作:
  - スレッド内に所属コミュニティへのリンク（パンくずり）が表示される。
  - リンクをクリックすると `/communities/$communityId`（または `/c/$slug`）に遷移する。

## UC-POST-16: 返信コメントに Reddit 風 L 字コネクターが表示される（#746）

- 前提条件: 親コメントと子コメント（返信）の両方が存在する。
- ステップ:
  1. `/posts/$postId` を開く。
- 期待動作:
  - 返信コメント内に L 字型の線分（親コメントの左端から返信へ繋がる線）が表示される。

## UC-POST-17: 投稿詳細でコメント数が正しく表示される（#779）

- 前提条件: コメントが付いた post が存在する。
- ステップ:
  1. `/posts/$postId` を開く。
- 期待動作:
  - 「コメント N 件」の小見出しが表示され、N が実障の公開済みコメント数に一致する。
  - コメントが 0 件の post では「コメント N 件」の見出しが表示されない（UC-POST-02 と一致）。

## UC-POST-18: vote ミューテーション進行中は vote ボタンが disabled になる（#748）

- 前提条件: post またはコメントが存在する。
- ステップ:
  1. `/posts/$postId` を開く。
  2. post または任意のコメントの up vote または down vote ボタンをクリックする（API 応答前）。
- 期待動作:
  - vote ボタンがクリック中（ミューテーション進行中）は up/down ボタンが `disabled` 状態になり、重複クリックが受け付けられない。
  - ミューテーション完了後はボタンが再度有効化される。

## UC-POST-19: 返信を持つコメントのアバター下にスレッドコネクターが表示される（#796）

- 前提条件: 返信コメントを持つコメントが存在する。
- ステップ:
  1. `/posts/$postId` を開く。
- 期待動作:
  - 返信コメントを持つコメントのアバターの左下に縦線（スレッドコネクター）が表示される。
  - 返信コメントを持たないコメントには表示されない。

## UC-POST-20: コメントに共有ボタンが表示され、コメントへのパーマリンクをコピー／X でシェアできる（#775）

- 前提条件: スレッドにコメントが存在する。
- ステップ:
  1. `/posts/$postId` を開く。
  2. 任意のコメントの共有ボタンをクリックする。
- 期待動作:
  - パーマリンクのコピーまたは X シェアができるメニューが表示される。

## UC-POST-21: vote ウィジェットに表示される数字は up - down のネットスコアである（#856）

- 前提条件: スレッドに post とコメントが表示されている。
- ステップ:
  1. `/posts/$postId` を開く。
  2. post およびコメントの vote ウィジェット中央の数字を確認する。
- 期待動作:
  - 表示される数字は up vote 数から down vote 数を引いたネットスコア（score）である。
  - score が負値の場合はマイナス付きの数字（例: `-2`）が表示される。

## UC-POST-22: ページリロード後も post・コメントの vote 状態が塗りつぶし表示で復元される（#831）

- 前提条件: スレッドで post またはコメントに vote 済み。
- ステップ:
  1. `/posts/$postId` を開く。
  2. post または任意のコメントに vote する。
  3. ページをリロードする。
- 期待動作:
  - リロード後も vote 済みの vote ウィジェットが塗りつぶし表示で復元される。

## UC-POST-23: コメントの共有リンクを開くと該当コメントまで自動スクロールされる（#861）

- 前提条件: コメントが複数存在するスレッドがある。
- ステップ:
  1. コメントの共有ボタンをクリックし、コピーされた URL（`/posts/$postId#comment-$commentId` 形式）をブラウザで開く。
- 期待動作:
  - ページが読み込まれた後、URL ハッシュで指定されたコメントまで自動的にスクロールされる。
  - スクロール先のコメントが画面中央付近に表示される。

## UC-POST-24: ネストコメントに返信元の一行プレビューが表示される（#931）

- 前提条件: 親コメントへの返信コメントが存在するスレッドがある。
- ステップ:
  1. `/posts/$postId` を開く。
- 期待動作:
  - 返信コメント（`parent_comment_id != null`）の本文上部に、グレー背景の引用枠が表示される。
  - 引用枠には親コメントのテキスト冒頭（最大 40 文字）が表示され、41 文字以上の場合は末尾に「…」が付く。
  - 引用枠をクリックすると親コメントの位置までスムーズスクロールされる。
  - トップレベルコメント（`parent_comment_id == null`）には引用枠が表示されない。
