# 画面設計書: チャンネル詳細画面（ChannelScene）

> 関連: Issue #30 / Issue #108 / 設計書 `docs/design/issue-30.md`
> ADR-0003（client）・ADR-0006（OpenAPI 型共有）・ADR-0007（docs）

## 目的

チャンネル別ビュー（`/channels/$channelId`）で、そのチャンネルに属するメッセージ（社員の掛け合い）を
Slack 風のフラットなメッセージ一覧として表示する。ログイン済みユーザーはメッセージ投稿も可能。

## URL / ルート

| URL | 画面 | 認証 |
|-----|------|------|
| `/channels/:channelId` | ChannelScene | 不要（投稿は要認証） |

参照: [`docs/design/url-list.md`](./url-list.md)

## 認証要否

表示は認証不要。投稿フォーム（MessageInput）はログイン済み時のみ表示。

## レイアウト

```
┌────────────┬─────────────────────────────┐
│ サイドバー  │ #チャンネルラベル（ヘッダ）       │
│ (RootLayout)│ ─────────────────────────── │
│ ・チャンネル │ ハル   おはようございます！        │
│   一覧      │ ケン   おはよう。                │
│            │ メイ   よろしくお願いします。      │
│            │ ─────────────────────────── │
│            │ [ メッセージ入力（ログイン時）]   │
└────────────┴─────────────────────────────┘
```

## 状態一覧

| 状態 | 説明 | ストーリー |
|------|------|----------|
| メッセージあり | 社員の掛け合いを一覧表示 | `WithMessages` |
| 空 | メッセージが 0 件 | `Empty` |

## データ契約

- `GET /channels/:channelId/messages` → `MessageRecord[]`（MSW でモック）
- `GET /auth/me` → `AuthUser | null`（ログイン状態でメッセージ入力を出し分け）
- `useParams` で `channelId` を取得（TanStack Router）
