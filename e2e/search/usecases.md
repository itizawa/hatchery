# search — 投稿全文検索のユースケース

`/search` ページで title / text の ILIKE 部分一致検索を行い、最大 50 件・新着順で結果を表示する（#751）。
認証不要の公開ページ。各ユースケースの見出し（`## UC-SEARCH-NN: ...`）は `search.spec.ts` の `test.todo()` と 1:1 で対応する。

## UC-SEARCH-01: ヘッダーの検索アイコンから検索ページへ遷移できる

- 前提条件: 任意のページを閲覧中。
- ステップ:
  1. ヘッダーの虫眼鏡（🔍）アイコンをクリックする。
- 期待動作: `/search` へ遷移し、検索入力フォームが表示される。

## UC-SEARCH-02: キーワードを入力して Enter を押すと検索結果が表示される

- 前提条件: `/search` を開いている。タイトルまたは本文にヒットする投稿が存在する。
- ステップ:
  1. 検索フォームにキーワードを入力する。
  2. Enter キーを押す（またはフォームを送信する）。
- 期待動作:
  - URL が `/search?q=<キーワード>` に更新される。
  - ヒットした投稿が新着順で一覧表示される（最大 50 件）。
  - 「N 件の投稿が見つかりました」という件数表示が出る。

## UC-SEARCH-03: 検索結果の投稿カードをクリックするとスレッドページへ遷移できる

- 前提条件: 検索結果が表示されている。
- ステップ:
  1. 任意の投稿カードをクリックする。
- 期待動作: `/posts/$postId` に遷移し、その投稿のスレッド詳細が表示される。

## UC-SEARCH-04: キーワードにヒットする投稿が 0 件のとき「見つかりませんでした」が表示される

- 前提条件: `/search` を開いている。入力したキーワードにヒットする投稿が存在しない。
- ステップ:
  1. ヒットしないキーワードを入力して Enter を押す。
- 期待動作: 「「<キーワード>」に一致する投稿が見つかりませんでした。」というメッセージが表示される。

## UC-SEARCH-05: 検索クエリなしで /search を開くと案内テキストが表示される

- 前提条件: なし。
- ステップ:
  1. クエリパラメータなしで `/search` を直接開く。
- 期待動作: 「キーワードを入力して投稿を検索できます。」という案内テキストが表示される。

## UC-SEARCH-06: URL の q パラメータを直接書き換えると検索結果が切り替わる

- 前提条件: 検索結果が表示されている。
- ステップ:
  1. URL の `q=` パラメータを別のキーワードに変更してアクセスする。
- 期待動作: 新しいキーワードで再検索された結果が表示される。
