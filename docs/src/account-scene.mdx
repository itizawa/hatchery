# 画面設計書: アカウント設定画面（AccountScene）

> 関連: Issue #108 / ADR-0003（client）・ADR-0007（docs）

## 目的

アカウント設定画面（`/account`）。ログイン済みユーザーが自身のプロフィール（表示名・パスワード等）を
確認・更新できる画面。`useAuth()` フックが `/auth/me` を呼び出してユーザー情報を取得する。

## URL / ルート

| URL | 画面 | 認証 |
|-----|------|------|
| `/account` | AccountScene | 必須（`requireAuth`: 未ログインは `/login` へリダイレクト） |

全画面の URL（ルート）一覧は Storybook の「設計/画面URL一覧（ルーティング表）」ページ（正本: `docs/design/url-list.md`）を参照してください。

## 認証要否

認証必須。`accountRoute` の `beforeLoad: requireAuth` により、未ログイン（`/auth/me` が 401）
またはネットワークエラー時は `/login` へリダイレクトされる（`client/src/router.tsx` を正とする）。
ログイン済みの場合は現在のプロフィールがフォームに反映される。

## 状態一覧

| 状態 | 説明 | ストーリー |
|------|------|----------|
| ログイン済み | ユーザー情報がフォームに表示される | `Default` |
| 未ログイン | `/auth/me` が 401 → フォーム空 | `LoggedOut` |

## データ契約

- `GET /auth/me` → `AuthUser | null`（ログイン状態の確認・フォーム初期値取得）
- `PATCH /auth/me` → `AuthUser`（プロフィール更新）
