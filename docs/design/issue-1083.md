# 設計書: fix: ホームの「ようこそ」演出でコミュニティが6件を超えると新設コミュニティがチップ一覧に表示されなくなる (#1083)

## 1. 目的 / 背景

`client/src/components/WelcomeSection.tsx` の「Hatchery へようこそ」演出（初回訪問者向けの購読導線）は、コミュニティチップを `communities.slice(0, 6)` で先頭6件に固定表示している。`usePublicCommunities` は `created_at` 昇順（作成が古い順）でコミュニティを返すため、コミュニティ数が6件を超えると新しく作成されたコミュニティは常にチップ一覧から除外される。concept.md の「最初の5分」体験設計では、この演出は新設コミュニティを含む注目コミュニティへの購読導線であるべきで、コミュニティ数が増える設計（ADR-0030 の休眠スロットリング等）を踏まえると固定上限は将来にわたって新設コミュニティの発見性を損なう。

## 2. スコープ（やること / やらないこと）

### やること
- `WelcomeSection` のチップ選出ロジックを、コミュニティ数が表示上限を超えても新設コミュニティが恒常的に除外されない方式に変更する。
- 具体策: `created_at` 降順（新しい順）に並べ替えたうえで上限件数のチップを表示する（新着優先・#1083 補足に例示された対応の1つ）。表示上限は 6 → 8 に引き上げ、当面のコミュニティ数増加（7件時点）でも既存コミュニティの露出を大きく削らずに新設コミュニティを含められるようにする。
- `WelcomeSection.test.tsx` にコミュニティが6件（新表示上限8件も想定し8件超のケースを含む）を超える場合に最新コミュニティが表示されることを検証するテストを追加する。

### やらないこと
- 演出の見た目（チップ以外の表現）の変更は対象外（Issue 本文で明記）。
- ランダム/ローテーション選出は採用しない（テストが決定的でなくなるため、新着順優先という決定的な方式を採用）。
- 表示上限を撤廃する（無制限表示）は将来的なレイアウト崩れの懸念があるため採用しない。

## 3. 受け入れ条件（テストに落とせる粒度）

1. `communities` が表示上限件数を超えるとき、`created_at` が最も新しいコミュニティがチップ一覧に含まれること。
2. `communities` が表示上限件数を超えるとき、表示されるチップ数は表示上限件数（8件）を超えないこと。
3. 既存の挙動（見出し表示・チップのリンク先・コミュニティ0件時の表示・「コミュニティを探す」ボタン表示）が壊れていないこと（既存テストが引き続き緑であること）。
4. `pnpm turbo run build test lint` が緑であること。

## 4. 設計方針（アーキ・データ構造・主要モジュール）

- `client/src/components/WelcomeSection.tsx` 内で `communities` を `created_at` 降順にソートしたコピー配列を作り、その先頭 `WELCOME_CHIP_LIMIT`（=8）件をチップ表示する。
- 元の `communities` prop 配列は破壊的変更しない（`[...communities].sort(...)`）。
- 比較関数は `new Date(b.created_at).getTime() - new Date(a.created_at).getTime()` で降順ソート。
- 表示上限値はコンポーネント内のモジュールスコープ定数 `WELCOME_CHIP_LIMIT` として定義し、マジックナンバーの直書きを避ける。

## 5. 影響範囲 / 既存への変更（対象ワークスペース: client / server / common / docs）

- 対象ワークスペース: `client` のみ（`client/src/components/WelcomeSection.tsx`、`client/src/components/WelcomeSection.test.tsx`）。
- server / common への変更なし（API レスポンスの並び順は変更しない。クライアント側でのみ並び替えて表示する）。

## 6. テスト計画（TDDで書くテスト一覧）

- `WelcomeSection.test.tsx` に追加:
  - 「コミュニティが8件を超える場合、最新のコミュニティがチップ一覧に表示される」
  - 「コミュニティが8件を超える場合、表示されるチップ数が8件を超えない」
- 既存5テストは変更せず継続して緑であることを確認する。

## 7. リスク・未決事項

- 表示上限を8件に引き上げたことで、コミュニティ数がさらに増えた場合（9件以上）は最も古いコミュニティ（`hatchery` 等の基幹コミュニティ）がチップから外れうる。これは Issue 本文が許容する「新着順を優先する」方式の意図した挙動であり、本 Issue のスコープでは許容する。将来的にコミュニティ数が大幅に増える場合は別 Issue でローテーション表示等を検討する。
