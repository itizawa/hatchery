# 設計書: 投稿・コメントの時刻表示を「1週間未満は相対時刻」に拡張し N日前 表示を追加する (#1016)

## 1. 目的 / 背景

現状の `formatRelativeTime` は 24 時間を境に相対→絶対日付へ切り替える。
24〜168 時間（1〜6 日）の投稿は絶対日付になるため、直感的な鮮度が読み取りにくい。
相対表示の窓を **7 日（168 時間）** へ広げ、日単位の相対表現（`N日前`）を追加する。

## 2. スコープ（やること / やらないこと）

**やること**
- `formatRelativeTime` の閾値変更: 24 時間 → 7 日
- 24 時間以上 7 日未満の範囲に `N日前`（端数切り捨て）を追加
- テスト境界を更新
- `PostedTime.tsx` のコメント内「24時間」→「1週間（7日）」に更新

**やらないこと**
- 週・月・年単位の相対表現（`N週間前` 等）
- i18n（英語表記）
- 閾値のユーザー設定化
- `server` への変更

## 3. 受け入れ条件（テストに落とせる粒度で箇条書き）

1. 23 時間 59 分 → `23時間前`（既存の 24 時間未満の挙動を維持）
2. ちょうど 24 時間 → `1日前`
3. 24 時間超〜 6 日（端数切り捨て） → `N日前`（1〜6）
4. ちょうど 7 日 → 絶対日付 `YYYY/M/D`
5. 7 日超 → 絶対日付 `YYYY/M/D`（既存挙動を維持）
6. 不正 Date（NaN）・未来時刻フォールバック（既存エッジケース）は変えない
7. `PostedTime.tsx` のホバーツールチップ（絶対日時）は `N日前` 表示でも維持される

## 4. 設計方針（アーキ・データ構造・主要モジュール）

変更は `common/src/logic/formatRelativeTime.ts` の 1 関数に閉じる。

```
diffMs < 1_000         → たった今
diffSeconds < 60       → N秒前
diffMinutes < 60       → N分前
diffMs < DAY_MS        → N時間前
diffMs < WEEK_MS(新規) → N日前  ← 追加
else                   → YYYY/M/D（絶対日付）
```

- `WEEK_MS = 7 * DAY_MS` を定数追加
- `date-fns` の `differenceInDays` を import 追加（既に他関数は import 済み）

## 6. テスト計画（TDDで書くテスト一覧）

| ケース | 入力 | 期待出力 |
|--------|------|---------|
| 23 時間 59 分 | `now - 23h59m` | `23時間前` |
| ちょうど 24 時間 | `now - 24h` | `1日前` |
| 24 時間 1 秒 | `now - 24h0m1s` | `1日前` |
| ちょうど 6 日 | `now - 6d` | `6日前` |
| 6 日 23 時間 59 分 | `now - 6d23h59m` | `6日前` |
| ちょうど 7 日 | `now - 7d` | `2026/6/7`（絶対日付） |
| 7 日超 | `now - 7d0m1s` | `2026/6/7`（絶対日付） |
| 旧テスト修正 | formerly `2026/6/13` for 24h → `1日前` | `1日前` |
