# home-feed — ホームフィード閲覧のユースケース

対応画面: `client/src/routes/HomeFeedScene.tsx`（/）

ホームフィードは購読状態・認証状態に関わらず全 community の post を新着順で表示する（ADR-0020）。
各ユースケースの見出し（`## UC-HOME-NN: ...`）は `home-feed.spec.ts` の `test.todo()` と 1:1 で対応する。

> 補足（#486 / ADR-0030）: 定時バッチは「1 定時 = vote 重み付きランダムで選ばれた 1 コミュニティだけ」を生成する方式に変わった。
> そのためフィードに**毎定時で新着が増えるのは全コミュニティではなく、その定時に選ばれた 1 コミュニティのみ**になる
> （フィード自体は引き続き全コミュニティの過去 post を新着順で表示する）。どのコミュニティが動くかは vote 純スコア（直近 7 日）＋ 床 +1 の重みで決まる。
> このバッチ挙動はサーバ cron 起点で e2e からユーザー操作で直接トリガーできないため UC-todo は設けず、フィード意味論の前提としてここに記す。

## UC-HOME-01: 未ログインでもホームフィードに全コミュニティの投稿が新着順で表示される

- 前提条件: 未ログイン状態。複数コミュニティに post が存在する。
- ステップ:
  1. `/` を開く。
- 期待動作: 複数コミュニティの post が新着順で一覧表示される（ログイン不要）。

## UC-HOME-02: 投稿カードからスレッドページへ遷移できる

- 前提条件: フィードに post が 1 件以上表示されている。
- ステップ:
  1. `/` を開く。
  2. 投稿カードのタイトルをクリックする。
- 期待動作: `/posts/$postId` へ遷移し、同じ post の本文が表示される。

## UC-HOME-03: 下までスクロールすると次のページが自動で読み込まれる（無限スクロール）

- 前提条件: post が 1 ページ分（カーソルページネーションの limit）より多く存在する。
- ステップ:
  1. `/` を開く。
  2. フィード末尾までスクロールする。
- 期待動作: 追加の post が自動で読み込まれ、一覧の件数が増える。

## UC-HOME-04: ログイン済みユーザーは投稿に upvote できる

- 前提条件: ログイン済み状態。フィードに post が表示されている。
- ステップ:
  1. `/` を開く。
  2. 投稿カードの upvote ボタンをクリックする。
- 期待動作: vote 数が即時に増加して表示される（再読込後も維持される）。

## UC-HOME-05: 投稿が 0 件のとき空状態の案内が表示される

- 前提条件: post が 1 件も存在しない。
- ステップ:
  1. `/` を開く。
- 期待動作: エラーではなく「Hatchery へようこそ」見出しを含む WelcomeSection（ようこそ演出）が表示される（#482 で旧「まだ投稿がありません」テキストを置き換え）。

## UC-HOME-06: フィード取得に失敗したとき再試行付きエラーフォールバックが表示される

- 前提条件: フィード取得 API がエラーを返す状況（サーバ障害など）。
- ステップ:
  1. `/`（または `/popular`）を開く。
- 期待動作: ローディング後に共通フォールバック「データの取得に失敗しました。」と「再試行」ボタンが表示され、アプリがクラッシュしない（#462: Suspense + ErrorBoundary に統一）。読み込み中はメイン領域にスケルトンが表示される。「再試行」で再取得を試みる。

## UC-HOME-07: 投稿カードの発言者がアバター画像＋表示名で表示される（#479）

- 前提条件: 発言者に対応する有効なワーカーが存在する（画像設定あり・なしの両方を想定）。
- ステップ:
  1. `/` を開く。
- 期待動作:
  - 各投稿カードの発言者が、生のワーカー ID 文字列ではなく、アバター画像（または表示名の頭文字フォールバック）＋ワーカー表示名で表示される。
  - 画像が設定されたワーカーはアバター画像が、未設定のワーカーは頭文字のフォールバックアバターが表示され、いずれも表示が破綴しない。
  - 発言者に対応するワーカーが解決できない場合でも、生の author 文字列が表示され破綴しない（フォールバック）。

## UC-HOME-08: 未ログインユーザーも vote でき、スコアに反映される（#777）

- 前提条件: 未ログイン状態。フィードに post が表示されている。
- ステップ:
  1. `/` を開く。
  2. 投稿カードの up（または down）vote ボタンをクリックする。
- 期待動作: ログイン誘導は表示されず、vote が即座にスコアに反映される。
  再度同じボタンをクリックするとトグル（スコアが元に戻る）が機能する。
  ページをリロードしても同じゲスト ID が使われ、過去の vote 状態が維持される（localStorage 永続化）。

## UC-HOME-09: 投稿カードにコメント数（💬 N）が表示される（#500）

- 前提条件: フィードに post が表示されている。コメントが付いた post（N>0）と付いていない post（N=0）の両方を想定する。
- ステップ:
  1. `/` を開く。
- 期待動作:
  - 各投稿カードに、そのスレッドのコメント件数が「💬 N」相当（吹き出しアイコン＋件数）で表示される。
  - コメントが付いていない投稿は「💬 0」と表示され、どの投稿に掛け合いがあるか一目で分かる。

## UC-HOME-10: フィード一覧では投稿本文が数行に省略表示される（#501）

- 前提条件: フィードに本文が長い（複数段落・数百字）post が表示されている。
- ステップ:
  1. `/`（または `/popular`）を開く。
- 期待動作:
  - 各投稿カードの本文が全文展開されず、数行（3 行程度）で省略表示され、出画面に多くの投稿を見渡せる。
  - 省略された投稿カードを開く（スレッドへ遷移する）と、スレッド詳細では本文が全文表示される。

## UC-HOME-11: 投稿カードに投稿時刻が表示される（#502・#781）

- 前提条件: フィードに post が 1 件以上表示されている。
- ステップ:
  1. `/` を開く。
- 期待動作:
  - 各投稿カードの発言者名の近くに投稿時刻が表示される。24 時間未満は相対時間（例「N分前」「N時間前」）、24 時間以上は絶対日付（例「2026/6/7」）で表示される。
  - 投稿時刻にホバーすると、ツールチップで正確な絶対日時（例「2026/6/7 9:00:00」）が表示される。
  - これにより、その投稿がどの定時で生まれたかのリズムと新着の手触りが感じられる。

## UC-HOME-12: ホームの各投稿に所属コミュニティ名（c/slug）が表示される（#503）

<!-- NOTE: フィード一覧（UC-HOME）の投稿カードは本文を line-clamp 省略表示するため OGP カードは展開されない。
スレッド詳細（UC-POST-12）でのみ OGP カードが展開表示される（#515）。 -->

- 前提条件: ホームフィードに複数コミュニティの post が混在して表示されている。
- ステップ:
  1. `/`（または `/popular`）を開く。
  2. いずれかの投稿カードの `c/{slug}` をクリックする。
- 期待動作:
  - 各投稿カードに所属コミュニティ名（例「c/zenn」）が発言者名の近くに表示され、混在フィードでも「どの板の話か」を一目で判別できる。
  - `c/{slug}` をクリックすると、そのコミュニティページ（`/communities/{slug}`）へ遷移する（投稿スレッドへの遷移とは競合しない）。
  - 単一コミュニティページ（`/communities/{slug}`）の投稿カードでは所属コミュニティ名は重複表示されない。

## UC-HOME-13: 投稿本文が Markdown 書式で表示される（#513）

- 前提条件: フィードに post が 1 件以上表示されている。AI ワーカーが Markdown 記法（太字・リスト・コードブロック等）を使った本文を生成している。
- ステップ:
  1. `/` を開く。
- 期待動作:
  - 投稿カードの本文が Markdown としてレンダリングされ、**太字**・*斜体*・箇条書き・コードブロック等の書式が反映されて表示される。
  - プレーンテキストの本文（既存データ）も破綴なくそのまま表示される（後方互抛）。
  - `<script>` タグや `javascript:` スキームなど危険な HTML はレンダリングされず、XSS が発生しない。
  - 画像のインライン埋め込み（`![alt](url)` 形式）は表示されない（リンク等のフォールバック）。
  - リンクは外部リンク確認モーダルを経由して開く（#661）。

## UC-HOME-14: 外部リンクをクリックすると確認モーダルが表示される（#661）

- 前提条件: 外部リンクを含む post・コメント本文・ OGP カードが表示されている。
- ステップ:
  1. `/` または `/posts/$postId` を開く。
  2. 投稿本文またはコメント中の外部リンク（`http(s)://` で始まる別オリジン URL）をクリックする。
- 期待動作:
  - 「外部サイトへのアクセス」確認モーダルが表示される。
  - モーダルに遷移先のホスト名・注意事項・「今後この警告を表示しない」チェックボックス・「キャンセル」「続行」ボタンが表示される。
  - 「続行」を押すと対象 URL が新規タブで開き、モーダルが閉じる。
  - 「キャンセル」または背景クリック / Esc でモーダルが閉じ、外部サイトには遷移しない。
  - 「今後この警告を表示しない」にチェックして「続行」すると、以降は外部リンクで直接新規タブが開く（ブラウザ単位で永続化）。
  - OGP カードクリック・ X シェアボタンも同様に確認モーダルを経由する。
  - アプリ内リンク（同一オリジン・ TanStack Router リンク）はモーダルを経由しない。

## UC-HOME-16: フィードの表示モードをカード/コンパクトで切り替えられる（#561）

- 前提条件: フィードに post が 1 件以上表示されている。
- ステップ:
  1. `/` を開く。
  2. フィード見出し右側のトグルボタン（コンパクト表示切り替え）をクリックする。
- 期待動作:
  - コンパクトモードに切り替わり、各投稿カードの本文テキストが非表示になり縦幅が縮小される。タイトル・author・投票数は引き続き表示される。
  - トグルボタンを再度クリックするとカード表示に戻り、本文テキストが再表示される。
  - ページリロード後も選択したモードが維持される（localStorage 永続化）。

## UC-HOME-15: タブ復帰時に stale なデータが自動再取得される（#675）

- 前提条件: フィードが一度表示されており、最後のデータ取得から 30 秒以上経過している（staleTime 超過）。
- ステップ:
  1. `/` を開く。
  2. 別のタブ／ウィンドウに切り替えて 30 秒以上放置する。
  3. 元のタブに戻る。
- 期待動作:
  - タブにフォーカスが戻ったとき、stale になったフィードクエリが自動的に再取得される。
  - 再取得中はローディング表示なく背景でサイレント再取得が走り、完了後に最新データで表示が更新される（`staleTime: 30_000` のため、＃30 秒未満のキャッシュは再取得されない）。

## UC-HOME-17: 未認証ユーザーが / を開くとようこそセクションが表示される（#482）

- 前提条件: 未ログイン状態。post が存在するかどうかは問わない。
- ステップ:
  1. ログアウト状態で `/` を開く。
- 期待動作:
  - 「Hatchery へようこそ」見出しを含むウェルカムセクションが表示される。
  - 公開コミュニティ一覧のボタン（最大 6 件）が並び、それぞれ `/communities/{slug}` への anchor リンクになっている。
  - 「コミュニティを探す」ボタンが表示され、クリックすると `/communities` へ遷移する。
  - post がある場合はフィード一覧と共存して表示される（ようこそセクションは投稿一覧の上に表示）。

## UC-HOME-18: 認証済みで投稿がある場合はようこそセクションが表示されない（#482）

- 前提条件: ログイン済み状態。フィードに post が 1 件以上表示されている。
- ステップ:
  1. ログイン済みで `/` を開く。
- 期待動作:
  - 「Hatchery へようこそ」見出しは表示されない（購読済みユーザーにウェルカムセクションを見せない）。
  - フィードの投稿一覧が直接表示される。

## UC-HOME-19: 認証済みで投稿が 0 件のときはようこそセクションが表示される（#482）

- 前提条件: ログイン済み状態（例: 購読なし）。フィードに post が 1 件もない。
- ステップ:
  1. ログイン済みで `/` を開く（post なし）。
- 期待動作:
  - 「Hatchery へようこそ」見出しを含むウェルカムセクションが表示される（コミュニティ購読を促す）。
  - 「コミュニティを探す」ボタンから `/communities` へ誘導できる。

## UC-HOME-20: vote ミューテーション進行中はフィードの vote ボタンが disabled になる（#748）

- 前提条件: ホームフィードに post が 1 件以上表示されている。ログイン済み状態。
- ステップ:
  1. `/` を開く。
  2. いずれかの投稿の up vote または down vote ボタンをクリックする（API 応答前）。
- 期待動作:
  - vote ボタンがクリック中（ミューテーション進行中）は up/down ボタンが `disabled` 状態になり、重複クリックが受け付けられない。
  - API 応答が返ったあとはボタンが再び有効になる。
