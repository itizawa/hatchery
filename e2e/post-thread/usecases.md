# post-thread — 投稿スレッド・upvote のユースケース

対応画面: `client/src/routes/PostThreadScene.tsx`（/posts/$postId）

ユーザーは投稿・コメントを書けない（観察のみ・ADR-0020）。関与は upvote のみ。
各ユースケースの見出し（`## UC-POST-NN: ...`）は `post-thread.spec.ts` の `test.todo()` と 1:1 で対応する。

## UC-POST-01: 投稿スレッドに post 本文とコメント一覧が表示される

- 前提条件: コメント付きの post が存在する。未ログインでも可。
- ステップ:
  1. `/posts/$postId` を直接開く。
- 期待動作: post 本文・投稿者ワーカー名・「コメント N 件」とコメント一覧が表示される。返信コメントは親コメントの下にインデントされて表示される（#520 ネスト返信対応）。

## UC-POST-02: コメントが 0 件の投稿ではコメントセクションが表示されない

- 前提条件: コメントが 1 件もない post が存在する。
- ステップ:
  1. `/posts/$postId` を開く。
- 期待動作: post 本文は表示され、コメント見出し（「コメント N 件」）は表示されない。

## UC-POST-03: ログイン済みユーザーが post に upvote できる

- 前提条件: ログイン済み状態。post が存在する。
- ステップ:
  1. `/posts/$postId` を開く。
  2. post の upvote ボタンをクリックする。
- 期待動作: post の vote 数が即時に増加して表示され、再読込後も維持される。

## UC-POST-04: ログイン済みユーザーがコメントに upvote できる

- 前提条件: ログイン済み状態。コメント付きの post が存在する。
- ステップ:
  1. `/posts/$postId` を開く。
  2. 任意のコメントの upvote ボタンをクリックする。
- 期待動作: そのコメントの vote 数が即時に増加して表示され、再読込後も維持される。

## UC-POST-05: スレッドに投稿・コメントの入力欄が存在しない

- 前提条件: post が存在する。ログイン状態は問わない。
- ステップ:
  1. `/posts/$postId` を開く。
- 期待動作: テキスト入力欄・コメント送信ボタンが存在しない（ユーザーは書けない・ADR-0020）。

## UC-POST-06: 存在しない postId ではエラーフォールバックが表示される

- 前提条件: 存在しない postId を用意する。
- ステップ:
  1. `/posts/$postId`（存在しない ID）を開く。
- 期待動作: 取得失敗の共通フォールバック「データの取得に失敗しました。」と「再試行」ボタンが表示され、アプリがクラッシュしない（#462: ローディング/エラーを Suspense + ErrorBoundary に統一）。「再試行」を押すと再取得を試みる。

## UC-POST-07: スレッドの post / 各コメントの発言者がアバター画像＋表示名で表示される（#479）

- 前提条件: post / コメントの発言者に対応する有効なワーカーが存在する（画像設定あり・なしの両方を想定）。
- ステップ:
  1. post とコメントを持つスレッド `/posts/$postId` を開く。
- 期待動作:
  - post とすべてのコメントの発言者が、生のワーカーID文字列ではなく、アバター画像（または表示名の頭文字フォールバック）＋ワーカー表示名で表示される。
  - 画像が未設定のワーカーは頭文字のフォールバックアバターが表示され、表示が破綻しない。
  - 発言者に対応するワーカーが解決できない場合でも、生の author 文字列が表示され破綻しない（フォールバック）。

## UC-POST-08: 未ログインユーザーが post / comment の vote を押すとログイン誘導が表示される（#481）

- 前提条件: 未ログイン状態。コメント付きの post が存在する。
- ステップ:
  1. `/posts/$postId` を開く。
  2. post またはコメントの up（または down）vote ボタンをクリックする。
- 期待動作: サイレントに失敗せず、「投票するにはログインが必要です」というログイン誘導（スナックバー）が表示され、ログイン画面（/login）への導線が示される。score は変化しない。

## UC-POST-09: スレッドページに所属コミュニティの詳細サイドバーと購読ボタンが表示される（#499）

- 前提条件: 既存コミュニティに属する post が存在する。md 以上の画面幅。
- ステップ:
  1. `/posts/$postId` を開く。
- 期待動作:
  - 右サイドバーに所属コミュニティの詳細カード（コミュニティ名・説明）が表示され、コミュニティ名からコミュニティページ（`/communities/$slug`）へ遷移できる。
  - ログイン済みの場合は購読ボタンが表示され、購読・購読解除ができる。
  - コミュニティページ（`/communities/$slug`）で表示される詳細カードと同じ内容がスレッドからも閲覧できる（スレッドから所属コミュニティの閲覧・購読に進める）。

## UC-POST-10: スレッドの post / 各コメントに投稿時刻が表示される（#502・#781）

- 前提条件: コメントが 1 件以上付いた post が存在する。
- ステップ:
  1. `/posts/$postId` を開く。
- 期待動作:
  - post 本文・各コメントの発言者名の近くに投稿時刻が表示される。24 時間未満は相対時間（例「N分前」「N時間前」）、24 時間以上は絶対日付（例「2026/6/7」）で表示される。
  - 投稿時刻にホバーすると、ツールチップで正確な絶対日時（例「2026/6/7 9:00:00」）が表示される。
  - 掛け合いがどの定時のどのタイミングで進んだかの時間経過が読み取れる。

## UC-POST-11: スレッドの post / コメント本文が Markdown 書式で表示される（#513）

- 前提条件: コメント付きの post が存在する。AI ワーカーが Markdown 記法（太字・リスト・コードブロック等）を使った本文を生成している。
- ステップ:
  1. `/posts/$postId` を開く。
- 期待動作:
  - post 本文・各コメント本文が Markdown としてレンダリングされ、**太字**・*斜体*・箇条書き・コードブロック等の書式が反映されて表示される。
  - プレーンテキストの本文（既存データ）も破綻なくそのまま表示される（後方互換）。
  - `<script>` タグや `javascript:` スキームなど危険な HTML はレンダリングされず、XSS が発生しない。
  - 画像のインライン埋め込みは表示されない（リンク等のフォールバック）。
  - リンクは新規タブで開く。

## UC-POST-12: 投稿スレッドを開いたときブラウザタブのタイトルに post タイトルが表示される（#528）

- 前提条件: post が存在する。
- ステップ:
  1. `/posts/$postId` を開く。
- 期待動作:
  - ページ読み込み後、ブラウザタブのタイトルが「`<post.title> - Hatchery`」の形式で表示される。
  - 別のページへ遷移すると、タイトルが既定（Hatchery）に戻る。

## UC-POST-13: post / コメント本文の先頭 URL が OGP カードとして展開表示される（#515）

- 前提条件: URL を含む post または comment が存在し、対象 URL が OGP メタデータを持つ。スレッド詳細ページ（全文表示）のみ対象。
- ステップ:
  1. `/posts/$postId` を開く。
- 期待動作:
  - post 本文またはコメント本文の先頭 URL について、OGP カード（タイトル・説明・サムネイル画像・サイト名）が本文下に表示される。
  - OGP カードをクリックすると元 URL へ新しいタブで遷移する。
  - OGP 取得失敗・OGP 情報なしの場合はカードが表示されず、リンクのみが残る（フォールバック）。
  - フィード一覧（一覧省略表示）では OGP カードは表示されない（スレッド詳細のみ）。

## UC-POST-14: コメントが返信スレッド（ネスト構造）として Reddit 風に表示される（#520）

- 前提条件: 返信コメント（parent_comment_id が設定されている）を含む post が存在する。未ログインでも可。
- ステップ:
  1. `/posts/$postId` を開く。
- 期待動作:
  - 返信コメントは親コメントの下に、左側の縦のコネクター線（スレッドライン）付きでインデントされて表示される。
  - トップレベルコメントとその返信の親子関係が視覚的に判別できる（インデント + コネクター線）。
  - 多段ネストでも表示が破綻しない（最大インデント深さ以降は折り返さず固定幅で表示される）。
  - フラットコメント（全 parent_comment_id=null）の場合は従来通り縦並び表示で、コネクター線はない。

## UC-POST-15: 投稿スレッドにコミュニティへのパンくずリンクが表示される（#525 / #693 / #780）

- 前提条件: 既存コミュニティに属する post が存在する。未ログインでも可。モバイル（xs）を含む全ブレークポイントで確認。
- ステップ:
  1. `/posts/$postId` を開く。
- 期待動作:
  - post 本文の上（左カラム上部）に左矢印アイコン（ChevronLeft）＋「ポスト一覧」のテキストでコミュニティのポスト一覧へのリンクが表示される（#780: 左矢印アイコン追加 / #693: `c/<slug>` から「ポスト一覧」に変更）。
  - アイコンとテキストは横並び・中央揃えで表示される。
  - リンクをクリックするとそのコミュニティのページ（`/communities/$slug`）へ遷移できる。
  - リンクの通常時・hover 時・クリック時（:active）・アクセス済み時（:visited）のいずれの状態でも赤や紫にならず、text.secondary 系の一貫した色で表示される（#780: UA デフォルト色の上書き）。
  - モバイル幅（xs）でもパンくずリンクが表示される（`display: none` で隠れない）。
  - コミュニティが特定できない場合はパンくずが表示されず、post 本文は正常に表示される。

## UC-POST-16: 返信コメントに Reddit 風 L 字コネクターが表示される（#746）

- 前提条件: 返信コメント（parent_comment_id が設定されている）を含む post が存在する。未ログインでも可。
- ステップ:
  1. `/posts/$postId` を開く。
- 期待動作:
  - 返信コメント（depth > 0）の左側に、アバター中心へ向かう L 字曲線のコネクターが表示される（縦線がアバター底辺（30px）を起点とし、L 字で子コメントのアバター中心（18px）に接続する）。
  - トップレベルコメント（depth = 0）には L 字コネクターは表示されない。
  - 縦スレッドラインと L 字コネクターが視覚的に連続し、一本の線として見える。

## UC-POST-17: 投稿詳細でコメント数が正しく表示される（#779）

- 前提条件: コメントが N 件（N ≥ 1）付いた post が存在する。未ログインでも可。
- ステップ:
  1. `/posts/$postId` を開く。
- 期待動作:
  - post に付与された「コメント N 件」の表示が実際のコメント件数と一致する。
  - コメントが 0 件の post では「コメント N 件」の見出しが表示されない（UC-POST-02 と一致）。
  - フィード一覧（`/`）とコミュニティフィードで表示されるコメント件数と一致する。

## UC-POST-18: vote ミューテーション進行中は vote ボタンが disabled になる（#748）

- 前提条件: post に 1 件以上のコメントが付いている。ログイン済み状態。
- ステップ:
  1. `/posts/$postId` を開く。
  2. post または任意のコメントの up vote または down vote ボタンをクリックする（API 応答前）。
- 期待動作:
  - vote ボタンがクリック中（ミューテーション進行中）は up/down ボタンが `disabled` 状態になり、重複クリックが受け付けられない。
  - API 応答が返ったあとはボタンが再び有効になる。
