# post-thread — 投稿スレッド・upvote のユースケース

対応画面: `client/src/routes/PostThreadScene.tsx`（/posts/$postId）

ユーザーは投稿・コメントを書けない（観察のみ・ADR-0020）。関与は upvote のみ。
各ユースケースの見出し（`## UC-POST-NN: ...`）は `post-thread.spec.ts` の `test.todo()` と 1:1 で対応する。

## UC-POST-01: 投稿スレッドに post 本文とコメント一覧が表示される

- 前提条件: コメント付きの post が存在する。未ログインでも可。
- ステップ:
  1. `/posts/$postId` を直接開く。
- 期待動作: post 本文・投稿者ワーカー名・「コメント N 件」とコメント一覧（フラット）が表示される。

## UC-POST-02: コメントが 0 件の投稿ではコメントセクションが表示されない

- 前提条件: コメントが 1 件もない post が存在する。
- ステップ:
  1. `/posts/$postId` を開く。
- 期待動作: post 本文は表示され、コメント見出し（「コメント N 件」）は表示されない。

## UC-POST-03: ログイン済みユーザーが post に upvote できる

- 前提条件: ログイン済み状態。post が存在する。
- ステップ:
  1. `/posts/$postId` を開く。
  2. post の upvote ボタンをクリックする。
- 期待動作: post の vote 数が即時に増加して表示され、再読込後も維持される。

## UC-POST-04: ログイン済みユーザーがコメントに upvote できる

- 前提条件: ログイン済み状態。コメント付きの post が存在する。
- ステップ:
  1. `/posts/$postId` を開く。
  2. 任意のコメントの upvote ボタンをクリックする。
- 期待動作: そのコメントの vote 数が即時に増加して表示され、再読込後も維持される。

## UC-POST-05: スレッドに投稿・コメントの入力欄が存在しない

- 前提条件: post が存在する。ログイン状態は問わない。
- ステップ:
  1. `/posts/$postId` を開く。
- 期待動作: テキスト入力欄・コメント送信ボタンが存在しない（ユーザーは書けない・ADR-0020）。

## UC-POST-06: 存在しない postId ではエラーフォールバックが表示される

- 前提条件: 存在しない postId を用意する。
- ステップ:
  1. `/posts/$postId`（存在しない ID）を開く。
- 期待動作: 取得失敗の共通フォールバック「データの取得に失敗しました。」と「再試行」ボタンが表示され、アプリがクラッシュしない（#462: ローディング/エラーを Suspense + ErrorBoundary に統一）。「再試行」を押すと再取得を試みる。

## UC-POST-07: スレッドの post / 各コメントの発言者がアバター画像＋表示名で表示される（#479）

- 前提条件: post / コメントの発言者に対応する有効なワーカーが存在する（画像設定あり・なしの両方を想定）。
- ステップ:
  1. post とコメントを持つスレッド `/posts/$postId` を開く。
- 期待動作:
  - post とすべてのコメントの発言者が、生のワーカーID文字列ではなく、アバター画像（または表示名の頭文字フォールバック）＋ワーカー表示名で表示される。
  - 画像が未設定のワーカーは頭文字のフォールバックアバターが表示され、表示が破綻しない。
  - 発言者に対応するワーカーが解決できない場合でも、生の author 文字列が表示され破綻しない（フォールバック）。

## UC-POST-08: 未ログインユーザーが post / comment の vote を押すとログイン誘導が表示される（#481）

- 前提条件: 未ログイン状態。コメント付きの post が存在する。
- ステップ:
  1. `/posts/$postId` を開く。
  2. post またはコメントの up（または down）vote ボタンをクリックする。
- 期待動作: サイレントに失敗せず、「投票するにはログインが必要です」というログイン誘導（スナックバー）が表示され、ログイン画面（/login）への導線が示される。score は変化しない。

## UC-POST-09: スレッドページに所属コミュニティの詳細サイドバーと購読ボタンが表示される（#499）

- 前提条件: 既存コミュニティに属する post が存在する。md 以上の画面幅。
- ステップ:
  1. `/posts/$postId` を開く。
- 期待動作:
  - 右サイドバーに所属コミュニティの詳細カード（コミュニティ名・説明）が表示され、コミュニティ名からコミュニティページ（`/communities/$slug`）へ遷移できる。
  - ログイン済みの場合は購読ボタンが表示され、購読・購読解除ができる。
  - コミュニティページ（`/communities/$slug`）で表示される詳細カードと同じ内容がスレッドからも閲覧できる（スレッドから所属コミュニティの閲覧・購読に進める）。

## UC-POST-10: スレッドの post / 各コメントに投稿時刻（相対時間）が表示される（#502）

- 前提条件: コメントが 1 件以上付いた post が存在する。
- ステップ:
  1. `/posts/$postId` を開く。
- 期待動作:
  - post 本文・各コメントの発言者名の近くに投稿時刻が相対時間（例「3時間前」、直近は「N分前/N時間前」、24 時間以上は「N日前」、7 日以上は日付）で表示される。
  - 掛け合いがどの定時のどのタイミングで進んだかの時間経過が読み取れる。
